          <h2>üìÅ Instance Format</h2>
          <p>
              Instances are stored in text files with the <code>.dat</code> extension.
          </p>

          <div class="spec-card">
              <h4>Naming Convention</h4>
              <p>The filename encodes the main characteristics of the instance:</p>
              <p><code>MPVRP_A_B_sC_dD_pE.dat</code></p>
          </div>

          <div class="spec-grid">
              <div class="spec-item">
                  <h5><span class="spec-badge">A</span> Category</h5>
                    <p>Instance type: L for large, M for medium, and S for small</p>
              </div>
              <div class="spec-item">
                  <h5><span class="spec-badge">B</span> Identifier</h5>
                  <p>Unique instance number (e.g., 001, 002)</p>
              </div>
              <div class="spec-item">
                  <h5><span class="spec-badge">C</span> Stations</h5>
                  <p>Number of customers to deliver</p>
              </div>
              <div class="spec-item">
                  <h5><span class="spec-badge">D</span> Depots</h5>
                  <p>Number of loading points</p>
              </div>
              <div class="spec-item">
                  <h5><span class="spec-badge">E</span> Products</h5>
                  <p>Number of different product types</p>
              </div>
          </div>

        <h3>Internal File Structure</h3>
        <p>The file is structured in sequential data blocks. Values are separated by spaces or tabs.</p>

        <div class="accordion">
            <div class="accordion-item active">
                <div class="accordion-header">
                    <span>1. UUID (Unique Identifier) - Line 1</span>
                    <svg class="icon" width="20" height="20" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                        <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"></path>
                    </svg>
                </div>
                <div class="accordion-content">
                    <p>The first line contains a unique v4 UUID generated for each instance:</p>
                    <pre># c01ab718-9a2c-4a7d-bb95-f37e2a389409</pre>
                    <p>This identifier ensures global uniqueness of the instance and allows tracing its origin.</p>
                </div>
            </div>

            <div class="accordion-item">
                <div class="accordion-header">
                    <span>2. Global Parameters (Line 2)</span>
                    <svg class="icon" width="20" height="20" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                        <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"></path>
                    </svg>
                </div>
                <div class="accordion-content">
                    <p>The second line defines the problem dimensions:</p>
                    <pre>NbProducts NbDepots NbGarages NbStations NbVehicles</pre>
                    <p><strong>Example:</strong> <code>3 2 1 20 5</code></p>
                    <ul>
                        <li>3 product types to distribute</li>
                        <li>2 depots (loading points)</li>
                        <li>1 garage (vehicle base)</li>
                        <li>20 service stations (customers)</li>
                        <li>5 available vehicles</li>
                    </ul>
                </div>
            </div>

        <div class="accordion-item">
            <div class="accordion-header">
                <span>3. Transition Cost Matrix</span>
                <svg class="icon" width="20" height="20" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                    <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"></path>
                </svg>
            </div>
            <div class="accordion-content">
                <p>
                    This square matrix (size <em>NbProducts √ó NbProducts</em>) defines the cleaning cost required to switch from one product to another in the truck's tank.
                </p>
                <pre>Cost_P1‚ÜíP1  Cost_P1‚ÜíP2  ...
Cost_P2‚ÜíP1  Cost_P2‚ÜíP2  ...
...</pre>
                <p>
                    The value at row <em>i</em> and column <em>j</em> is the cost to switch from product <em>i</em> to product <em>j</em>.
                    The diagonal is generally zero (no cost to keep the same product).
                </p>
            </div>
        </div>

            <div class="accordion-item">
                <div class="accordion-header">
                    <span>4. Vehicle Fleet</span>
                    <svg class="icon" width="20" height="20" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                        <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"></path>
                    </svg>
                </div>
                <div class="accordion-content">
                    <p>Each line describes an available vehicle:</p>
                    <pre>ID  Capacity  HomeGarage  InitialProduct</pre>
                    <table>
                        <tr><th>Field</th><th>Description</th></tr>
                        <tr><td><code>ID</code></td><td>Unique vehicle identifier</td></tr>
                        <tr><td><code>Capacity</code></td><td>Maximum transportable volume</td></tr>
                        <tr><td><code>HomeGarage</code></td><td>ID of the assigned garage</td></tr>
                        <tr><td><code>InitialProduct</code></td><td>Initial tank configuration</td></tr>
                    </table>
                </div>
            </div>

            <div class="accordion-item">
                <div class="accordion-header">
                    <span>5. Depots (loading points)</span>
                    <svg class="icon" width="20" height="20" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                        <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"></path>
                    </svg>
                </div>
                <div class="accordion-content">
                    <p>Each line defines a depot:</p>
                    <pre>ID  X  Y  Stock_P1  Stock_P2  ...  Stock_Pp</pre>
                    <ul>
                        <li><code>X, Y</code>: Geographic coordinates</li>
                        <li><code>Stock_Pi</code>: Available quantity for product <em>i</em></li>
                    </ul>
                </div>
            </div>

            <div class="accordion-item">
                <div class="accordion-header">
                    <span>6. Garages (bases)</span>
                    <svg class="icon" width="20" height="20" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                        <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"></path>
                    </svg>
                </div>
                <div class="accordion-content">
                    <p>Each line defines a garage:</p>
                    <pre>ID  X  Y</pre>
                    <p>Garages serve only as departure and arrival points for routes.</p>
                </div>
            </div>

            <div class="accordion-item">
                <div class="accordion-header">
                    <span>7. Service Stations (customers)</span>
                    <svg class="icon" width="20" height="20" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                        <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"></path>
                    </svg>
                </div>
                <div class="accordion-content">
                    <p>Each line defines a station and its needs:</p>
                    <pre>ID  X  Y  Demand_P1  Demand_P2  ...  Demand_Pp</pre>
                    <ul>
                        <li><code>Demand_Pi</code>: Required quantity for product <em>i</em></li>
                        <li>A demand of 0 means the station does not need this product.</li>
                    </ul>
                </div>
            </div>

          <h2>üìã Solution Format</h2>
          <p>
              To be validated, a solution must be provided in a text file strictly following the structure below.
              The filename must match the instance being solved (e.g., <code>Sol_MPVRP_...</code>) always with the <code>.dat</code> extension.
          </p>

          <h3>File Structure</h3>
          <p>The file describes the routes vehicle by vehicle, then ends with a performance summary.</p>

          <div class="accordion">
              <div class="accordion-item active">
                  <div class="accordion-header">
                      <span>Route Description</span>
                      <svg class="icon" width="20" height="20" fill="none" stroke="currentColor" viewBox="0 0 24 24">
                          <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M19 9l-7 7-7-7"></path>
                      </svg>
                  </div>
                  <div class="accordion-content">
                      <p>For each vehicle used, the solution contains a block of <strong>2 lines</strong>:</p>

                      <h5>Line 1: Visit Sequence</h5>
                      <pre>Garage - Depot - Station(Qty) - ... - Garage</pre>
                      <p>Describes the path taken. Quantities delivered to stations are indicated in parentheses.</p>

                      <h5>Line 2: Product Sequence and Costs</h5>
                      <pre>Prod(Cost) - Prod(Cost) - ...</pre>
                      <p>Indicates which product is transported at each step and the associated changeover cost (if applicable).</p>

                      <div class="info-box" style="margin-top: 15px; padding: 10px;">
                          <p style="font-size: 0.9em; margin: 0;">
                              <strong>Note:</strong> The two lines must be perfectly aligned in terms of the number of steps.
                          </p>
                      </div>
                  </div>
              </div>
          </div>

    <h3>Valid Solution Example</h3>
    <pre>1 - 3 - 8 ( 51 ) - 9 ( 63 ) - 1
3(0.0) - 3(0.0) - 3(0.0) - 3(0.0) - 3(0.0)

2 - 4 - 13 ( 18 ) - 2
2(0.0) - 2(0.0) - 2(0.0) - 2(0.0)
</pre>

        // Handle dropdown on mobile
        const dropdowns = document.querySelectorAll('.dropdown');
        dropdowns.forEach(dropdown => {
            const dropbtn = dropdown.querySelector('.dropbtn');
            dropbtn.addEventListener('click', function(e) {
                if (window.innerWidth <= 768) {
                    e.preventDefault();
                    dropdown.classList.toggle('active');
                }
            });
        });

        // Close mobile menu when clicking outside
        document.addEventListener('click', function(e) {
            if (!e.target.closest('nav')) {
                document.querySelector('.nav-links').classList.remove('active');
                dropdowns.forEach(dropdown => dropdown.classList.remove('active'));
            }
        });

        // Accordion functionality
        document.addEventListener('DOMContentLoaded', function() {
            const accordionHeaders = document.querySelectorAll('.accordion-header');

            accordionHeaders.forEach(header => {
                header.addEventListener('click', function() {
                    const accordionItem = this.parentElement;
                    const isActive = accordionItem.classList.contains('active');

                    // Close all accordion items
                    document.querySelectorAll('.accordion-item').forEach(item => {
                        item.classList.remove('active');
                    });

                    // If this item wasn't active, open it
                    if (!isActive) {
                        accordionItem.classList.add('active');
                    }
                });
            });
        });
