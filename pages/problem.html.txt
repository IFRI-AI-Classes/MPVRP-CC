            <h1>Problem Statement</h1>

            <p>
                The <strong>Multi-Product Vehicle Routing Problem with Changeover Cost</strong>
                (<em>Multi-Product Multi-Depot Vehicle Routing Problem with Changeover Cost - MPVRP-CC</em>)
                is a complex logistics optimization challenge. It aims to organize the efficient distribution of multiple product types
                (e.g., different fuels) from a set of depots to a network of service stations.
            </p>

            <img src="static/imgs/MPVRP-CO.jpg" alt="MPVRP-CC Illustration" class="img-fluid">

            <h2>Context and Definitions</h2>

            <h3>Mathematical Notation</h3>
            <p>The problem is modeled on the following sets:</p>
            <ul class="math-notation">
                <li><i>K</i>: Set of available trucks, <i>K</i> = {1, ..., <i>K</i>}</li>
                <li><i>P</i>: Set of products to distribute, <i>P</i> = {1, ..., <i>P</i>}</li>
                <li><i>G</i>: Set of garages (truck departure/arrival points), <i>G</i> = {1, ..., <i>G</i>}</li>
                <li><i>D</i>: Set of depots (loading points), <i>D</i> = {1, ..., <i>D</i>}</li>
                <li><i>S</i>: Set of service stations (customers), <i>S</i> = {1, ..., <i>S</i>}</li>
            </ul>

            <p>
                The logistics network connects these different sites. A distance matrix defines the separation between each pair of locations.
                Each service station expresses a specific demand for each product type.
                To meet this demand, a heterogeneous fleet of tanker trucks is deployed. Each vehicle has a defined loading capacity
                and is attached to a specific garage.
            </p>

            <h2>Operational Framework</h2>
            <p>
                Truck operations follow a rigorous structure: <strong>Garage → [Depot → Customers]... → Garage</strong>.
            </p>

            <h3>Route Structure</h3>
            <p>
                A <strong>complete route</strong> must start and end at the vehicle's assigned garage.
                It consists of a succession of <strong>mini-routes</strong>.
            </p>
            <p>
                A mini-route corresponds to a delivery cycle:
            </p>
            <ul>
                <li><strong>Loading:</strong> The truck goes to a depot to load one (and only one) product type.</li>
                <li><strong>Delivery:</strong> It then serves one or more service stations to deliver this product.</li>
                <li><strong>Return:</strong> Once empty or the route is complete, it returns to a depot to reload or goes back to its garage.</li>
            </ul>

            <h3>Multi-Product Management</h3>
            <p>
                The particularity of this problem lies in product management:
            </p>
            <ul>
                <li>A truck can only transport one product type at a time (single or dedicated compartment).</li>
                <li>Each truck is initially configured for a given product.</li>
                <li><strong>Changeover Cost:</strong> It is possible to change products during a visit to the depot.
                    However, this operation requires tank cleaning which incurs a specific cost.
                    The optimization must therefore balance between making detours to keep the same product or paying this cost to change products on site.</li>
            </ul>

            <h2>Objectives and Constraints</h2>

            <p>
                The objective of MPVRP-CC is to determine the routes for the entire fleet in order to <strong>minimize the total cost</strong>, composed of:
            </p>
            <ul>
                <li>Transportation cost (proportional to the total distance traveled).</li>
                <li>Total product changeover cost (tank cleaning).</li>
            </ul>

            <p>
                A valid solution must strictly respect the following constraints:
            </p>
            <ul>
                <li><strong>Demand satisfaction:</strong> All station demands, for all products, must be fully delivered.</li>
                <li><strong>Capacity:</strong> The loaded quantity must never exceed the truck's maximum capacity.</li>
                <li><strong>Flow:</strong> Each truck must end its day at its home garage.</li>
                <li><strong>Uniqueness:</strong> A truck does not serve the same station multiple times for the same product during a single mini-route.</li>
            </ul>

                    </svg>
                </div>
                <p>
                    <strong>Assumptions:</strong> We assume that all depots have sufficient stock to satisfy all demands and that all sites are accessible without time constraints.
                </p>
            </div>

                if (window.innerWidth <= 768) {

                    dropdown.classList.toggle('active');

        document.addEventListener('click', function(e) {
            if (!e.target.closest('nav')) {
                document.querySelector('.nav-links').classList.remove('active');
                dropdowns.forEach(dropdown => dropdown.classList.remove('active'));
            }
        });

        // Accordion functionality
        document.addEventListener('DOMContentLoaded', function() {
            const accordionHeaders = document.querySelectorAll('.accordion-header');

            accordionHeaders.forEach(header => {
                header.addEventListener('click', function() {
                    const accordionItem = this.parentElement;
                    const isActive = accordionItem.classList.contains('active');

                    // Close all accordion items
                    document.querySelectorAll('.accordion-item').forEach(item => {
                        item.classList.remove('active');
                    });

                    // If this item wasn't active, open it
                    if (!isActive) {
                        accordionItem.classList.add('active');
                    }
                });
            });
        });
